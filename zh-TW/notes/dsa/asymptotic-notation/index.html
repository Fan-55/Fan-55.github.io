Big-O Notation​

We use T(n)T(n)T(n), a function of input size nnn, to represent the bound on the worst-case running time of an algorithm. The meaning of T(n)=O(f(n))T(n) = O(f(n))T(n)=O(f(n)) for some canonical function f(n)f(n)f(n) like nnn, nlognnlognnlogn, or n2n^2n2 can be expressed as:

Big-O Notation

  • English Definition:

    T(n)=O(f(n))T(n) = O(f(n))T(n)=O(f(n)) if and only if T(n)T(n)T(n) is eventually bounded above by a constant multiple of f(n)f(n)f(n).

  • Mathematical Definition:

    T(n)=O(f(n))T(n) = O(f(n))T(n)=O(f(n)) if and only if there exist positive constants ccc and n0n_0n0​ such that T(n)≤c⋅f(n)T(n) \le c \cdot f(n)T(n)≤c⋅f(n) for all n≥n0n \ge n_0n≥n0​.

Mathematical definition is a direct translation of the English definition. The inequality in mathematical definition expresses that T(n)T(n)T(n) should be bounded above by a multiple of f(n)f(n)f(n) (with the constant ccc specifying the multiple). The "for all n≥n0n \ge n_0n≥n0​" expresses that the inequality only needs to hold eventually, once nnn is sufficiently large (with the constant n0n_0n0​ specifying how large).

If big-O notation is analogous to less than or equal to (≤\le≤), then big-omega and big-theta are analogous to greater than or equal to (≥\ge≥) and equal to (===) respectively.
