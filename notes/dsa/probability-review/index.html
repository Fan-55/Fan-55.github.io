1. Sample space​

The sample space Ω\OmegaΩ is a set of all possible outcomes of a random process. Ω\OmegaΩ is usually finite in algorithms. Each outcome i∈Ωi\in\Omegai∈Ω has a probability p(i)≥0p(i)\ge 0p(i)≥0 and the sum of the probabilities should be 111 (∑i∈Ωp(i)=1\sum\limits_{i\in\Omega}p(i)=1i∈Ω∑​p(i)=1).

Example 1:

  • Random process: Roll 2 dice.

    Ω={(1,1),(2,1),(3,1),...(5,6),(6,6)}\Omega=\{(1,1),(2,1),(3,1),...(5,6),(6,6)\}Ω={(1,1),(2,1),(3,1),...(5,6),(6,6)}.

  • Random process: Chose a random pivot in outermost quicksort call.

    Ω={1,2,...,n}\Omega=\{1,2,...,n\}Ω={1,2,...,n}

    The sample space is every index of input array of length nnn and p(i)=1np(i)=\dfrac 1 np(i)=n1​ for all i∈Ωi\in\Omegai∈Ω

2. Events​

An event is a subset of sample space S⊆ΩS\subseteq\OmegaS⊆Ω. An event is just a bunch of stuff that might happen. We know the probability of each individual thing that can happen, so we add them up the get the probability of an event. The probability of an event SSS is ∑i∈Sp(i)\sum\limits_{i\in S}p(i)i∈S∑​p(i).

Example 1: Rolling 2 dice. Consider the subset of outcomes in which the sum of the two dice equals 777. What is the probability of this event?

S={(1,6),(2,5),(3,4),(4,3),(5,2),(6,1)}S=\{(1,6),(2,5),(3,4),(4,3),(5,2),(6,1)\}S={(1,6),(2,5),(3,4),(4,3),(5,2),(6,1)}

Each of the 36 outcomes is equally likely and has the probability of 136\dfrac 1 {36}361​. We have 6 members of the set, so the probability Pr[S]=636=16Pr[S]=\dfrac 6 {36}=\dfrac 1 6Pr[S]=366​=61​.

Example 2: Consider the event (i.e., the subset of outcomes) in a quicksort call that the chosen pivot will give a 25-75 split or better (both of the subarrays have at least 25% of the elements). What's the probability of this event?

Let's assume the array has 100 elements. We want both subarrays to have at least 25% of the elements, so neither one has more than 75% of the elements. If we choose an element that's 26 or bigger in value, the left subarray will have 25 elements from 1 through 25. If we choose an element that's at most 75, the right subarray will have at least 25 elements from 76 through 100. Hence, everything between 26 and 75 inclusive is going to give us a 25-75 split. More generally, any pivot from the middle 50% of the quantiles is going to give us the desired split.

4. Expectation​

The expectation or expected value E[X]E[X]E[X] of a random variable XXX is its average value over everything that could happen (all the outcomes), weighted appropriately with the possibilities of different outcomes.

The value of random variable depends on the outcome of a random process. We can't predict what would be the outcome of a random process and thus we can't predict the value of a random variable. That's why it's random! That is also the reason why we would like to have a value to represent the value of random variable on average. If the random process occurs repeatedly, E[X]E[X]E[X] is the long-run average value of random variable XXX.

The mathematical expression of E[X]E[X]E[X]:

E[X]=∑i∈Ωp(i)⋅X(i)E[X]=\sum\limits_{i\in\Omega}p(i)\cdot X(i)E[X]=i∈Ω∑​p(i)⋅X(i)

where p(i)p(i)p(i) is the probability of outcome i∈Ωi\in\Omegai∈Ω.

Example 1:

  • Random process: Roll 2 dice.

  • Random variable: Sum of the two dice.

    There are 6∗6=366 * 6 = 366∗6=36 outcomes in the sample space.

    Ω={(1,1),(2,1),(3,1),...(5,6),(6,6)}\Omega=\{(1,1),(2,1),(3,1),...(5,6),(6,6)\}Ω={(1,1),(2,1),(3,1),...(5,6),(6,6)}

    The probability of each outcome is 136\dfrac 1{36}361​.

    The expectation of the sum of the two dice:

    E[X]=2⋅136+3⋅136+...+12⋅136=7E[X] = 2\cdot\dfrac1{36} + 3\cdot\dfrac1{36} + ... + 12\cdot\dfrac1{36} = 7E[X]=2⋅361​+3⋅361​+...+12⋅361​=7

5. Linearity of Expectation​

∑j=1nE[Xj]=∑j=1n∑i∈Ωp(i)⋅Xj(i)=∑i∈Ω∑j=1np(i)⋅Xj(i)=∑i∈Ωp(i)(∑j=1nXj(i))=E[∑j=1nXj(i)]\sum\limits_{j=1}^nE[X_j] \\= \sum\limits_{j=1}^n \sum\limits_{i\in\Omega} p(i)\cdot X_j(i) \\= \sum\limits_{i\in\Omega} \sum\limits_{j=1}^n p(i)\cdot X_j(i) \\= \sum\limits_{i\in\Omega}p(i) (\sum\limits_{j=1}^n X_j(i)) \\= E[\sum\limits_{j=1}^nX_j(i)]j=1∑n​E[Xj​]=j=1∑n​i∈Ω∑​p(i)⋅Xj​(i)=i∈Ω∑​j=1∑n​p(i)⋅Xj​(i)=i∈Ω∑​p(i)(j=1∑n​Xj​(i))=E[j=1∑n​Xj​(i)]

Example:

  • Random process of load balancing: Randomly assign a process to one of the nnn servers, and there are nnn processes to be sent to the server.

  • Random variable YYY: The count of processes assigned to any one of the nnn servers, which is also known as the server load. The story is the same for all the servers due to symmetry, so we may just focus on the first server – server 1.

  • Goal: Compute the expectation of server load E[Y]E[Y]E[Y] (i.e., the average server load).

    Since each process can be assigned to one of the nnn servers, the sample space has nnn^nnn outcomes. Each outcome would look like this:

    10___P1 5___P2 ... 79___Pn \underset{P_1}{\underset{\_\_\_}{10}}\:\underset{P_2}{\underset{\_\_\_}{5}}\:\underset{...}{}\:\underset{P_n}{\underset{\_\_\_}{79}}\:P1​___10​​P2​___5​​...​Pn​___79​​

    where PiP_iPi​ denotes process iii and the corresponding value is the id of server ranging from 111 to nnn. In the outcome above, process 1 is assigned to server 10, process 2 is sent to server 5, etc. Given an outcome, we can compute the count of processes that assigned to server 1, which is the value that random variable YYY can take on.

    Computing E[Y]E[Y]E[Y] using the definition of expectation is impractical since the number of outcomes is too big to handle. Luckily, YYY can be expressed as the sum of simpler random variables XjX_jXj​s.

    For the simpler random variable XjX_jXj​,

    • Random process: Randomly assign a process to one of the nnn servers once.

    • Random variable XjX_jXj​: The count of processes assigned to server 1 at jjjth time. jjj ranges from 111 to nnn.

      Because one process can be assigned to one of the nnn servers, the sample space Ω={1,2,...,n}\Omega=\{1,2,...,n\}Ω={1,2,...,n}. Each outcome is the id of the server that the process is possibly assigned to. Given an outcome, we can compute the count of processes that assigned to server 1, which is either 000 or 111.

      Xj={1if the jth process is assigned to server 10otherwiseX_j=\begin{cases} 1 & \text{if the jth process is assigned to server 1}\\ 0 & \text{otherwise} \end{cases}Xj​={10​if the jth process is assigned to server 1otherwise​

    Because the total count of processes assigned to server 1 is the sum of the count of processes assigned to server 1 per assignment, we get:

    (1) Y=∑j=1nXjY=\sum\limits_{j=1}^{n}{X_j}Y=j=1∑n​Xj​

    Combining the definition of linearity expectation E[∑j=1nXj]=∑j=1nE[Xj]E[\sum\limits_{j=1}^nX_j]=\sum\limits_{j=1}^nE[X_j]E[j=1∑n​Xj​]=j=1∑n​E[Xj​] and eq(1), we get:

    (2) E[Y]=eq(1)E[∑j=1nXj]=linearity expectation definition∑j=1nE[Xj]E[Y]\stackrel{eq(1)}{=}E[\sum\limits_{j=1}^{n}{X_j}]\stackrel{\text{linearity expectation definition}}{=}\sum\limits_{j=1}^nE[X_j]E[Y]=eq(1)E[j=1∑n​Xj​]=linearity expectation definitionj=1∑n​E[Xj​]

    Unlike computing E[Y]E[Y]E[Y], it's easy to compute the expectation of XjX_jXj​. By the definition of expectation:

    (3) E[Xj]=Pr[Xj=0]⋅0+Pr[Xj=1]⋅1=Pr[Xj=1]=1nE[X_j] = Pr[X_j=0] \cdot 0 + Pr[X_j=1] \cdot 1 = Pr[X_j=1] = \dfrac 1 nE[Xj​]=Pr[Xj​=0]⋅0+Pr[Xj​=1]⋅1=Pr[Xj​=1]=n1​
